.NET

在開發環境中，可以使用 MSBuild或 Visual Studio來組建程式。

  • 在使用 MSBuild時生成偵錯符號(Debug Symbol)，需要在組建中的屬性包含 DebugSymbols=true和 DebugType=Pdbonly。
  • 在使用Visual Studio 2019或更早版本時生成偵錯符號(Debug Symbol)，請於專案名稱右鍵單擊，選擇 Properties > Build > Advanced > Advanced Build Settings > Debugging Information設置為 Pdb-only。
  • 在使用Visual Studio 2022時生成偵錯符號(Debug Symbol)，對於.NET或.NET Core專案，請於專案名稱右鍵單擊，選擇Properties > Build > General > Advanced Build Settings > Debugging Information設置為PDB files, current platform；對於其他專案(如.NET Framework專案)，請將Debugging Information設置為Pdb-only。

如果應用程式是使用 ASP.NET web application 或 web site template 開發，請於開發環境執行Publish，發佈該應用程式，並將產生之資料夾一併放入欲進行掃描的專案資料夾根目錄。

  • 要使用Visual Studio 2019或更高的版本 發佈應用程式，請右鍵單擊Solution Explorer中的應用程式專案名稱，選擇Publish，然後選擇Publish至資料夾。在Publish設置中，點擊顯示所有設定，展開File Publish Options，選擇Precompile during publishing，然後點擊Configure，在Advanced Precompile Settings框中，取消勾選Allow precompiled site to be updatable，並勾選Emit debug information，然後選擇保存上述設定，最後選擇Publish以將應用程式發佈到資料夾。
  • 要使用Visual Studio 2017或更早的版本 發佈應用程式，請右鍵單擊Solution Explorer中的應用程式專案名稱，選擇Publish，在Publish設置中，選擇Settings的選項，展開File Publish Options，選擇Precompile during publishing，然後點擊Configure，在Advanced Precompile Settings框中，取消勾選Allow precompiled site to be updatable，並勾選Emit debug information，然後選擇保存上述設定，最後選擇Publish 以將應用程式發佈到資料夾。
  • 如欲使用ASP.NET Compilation Tool 發佈應用程式，請打開Developer PowerShell for Visual Studio，輸入aspnet_compiler.exe -p "C:\source\website" -v / -f "C:\published"，其中C:\source\website 是應用程式的根目錄，C:\published 是儲存發佈應用程式的任意位置，請依照您的環境實際狀況做修改。

組建完成後，請注意專案的Primary Build Artifact (如 .dll 或 .exe 檔案) 需位於專案資料夾之中，將之與您想掃描的Source Code 一併提供並告知我們其位置的資訊。

如果Build Artifact產生後在專案資料夾之外，請將Build Artifact複製到專案資料夾的目錄之中 (如: .\bin) 。

針對網站的專案，任何的 App_.dll 檔案 (如: *App_Code.dll ) 都可以做為Primary Build Artifact，我們將指定它為掃描專案的分析目標。

打包產生一個 .Zip 檔案提供我們掃描時，請注意專案的程式及檔案需直接放至 .Zip 的根目錄之中，不需另外放一層大的專案資料夾(Project Folder) 來含括各個相關的檔案資料夾。

Zip檔應包含專案檔案 (a .*proj file and/or web.config)、Source Code 和所有的 Build Artifacts.
